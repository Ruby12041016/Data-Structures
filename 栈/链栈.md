### 链栈（Linked Stack）

**链栈**是使用链表来实现的栈，栈的基本操作（入栈、出栈、栈顶）都是在链表的头部进行的。

#### 1. 链栈的基本操作

| 操作          | 描述                   |
| ----------- | -------------------- |
| `push()`    | 将一个元素压入栈中，通常发生在链表的头部 |
| `pop()`     | 从栈中弹出一个元素，通常发生在链表的头部 |
| `top()`     | 获取栈顶元素的值，但不删除它       |
| `isEmpty()` | 判断栈是否为空              |
| `size()`    | 获取栈中的元素个数            |
| `clear()`   | 清空栈                  |

#### 2. 链栈的实现

链栈通常使用链表来实现，其中每个节点包含两个部分：

* **数据部分**：存储栈中的数据元素。
* **指针部分**：指向下一个栈元素。


#### 3. 链栈的特点

* **动态内存分配**：链栈不像数组栈那样需要预先分配固定大小的内存空间，因此可以动态增长，避免了栈溢出问题。
* **灵活性高**：可以随时插入和删除元素，不需要进行元素的移动。

#### 4. 链栈的实现细节

* **栈顶指针（Top）**：指向链栈的栈顶元素。每次 `push` 操作时，新的节点会成为栈顶节点；每次 `pop` 操作时，栈顶元素会被删除。
* **链表节点**：每个节点有两个字段：数据域（存储元素）和指针域（指向下一个节点）。



#### 5. 时间复杂度

* `push()`、`pop()` 和 `top()` 操作的时间复杂度是 O(1)，因为这些操作都是在链表头部进行的。
* `isEmpty()` 和 `size()` 操作的时间复杂度也是 O(1)，其中 `size()` 可能需要一个额外的计数器来追踪栈中元素的个数。
